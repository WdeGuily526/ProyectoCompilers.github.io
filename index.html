Fases del proceso de un compilador

Análisis Léxico

Convierte el código fuente en un stream de caracteres que se convierten en un conjunto de lexemas con significados específicos que denominamos tokens.

Análisis Sintáctico

Toma los tokens producidos y genera un árbol de sintaxis, se lo revisa de acuerdo a la gramática del lenguaje definido y verifica que las expresiones sean sintacticamente correctas.

Análisis Semántico

Valida si el árbol sintáctico construido concuerda con las reglas del lenguaje formal y mantiene control de los identificadores con sus respectivas expresiones

Optimización y Genración de Código

Se genera un código intermedio, el cual se optimiza para dar una versión final que se mapea al lenguaje de máquina objetivo.

Compilador vs Interprete

Diferencia Importante

Tanto los compiladores como los intérpretes cumplen la función de convertir el código de software que se ha escrito a un formato ejecutable y legible por máquina. El intérprete traduce el código durante el tiempo de ejecución linea por linea, mientras que el compilador lo hace con todo el código antes de ejecutarlo.

Generador de código Intermedio

El generador de código intermedio crea una representación intermedia del programa que facilita las optimizaciones y la generación de código final. En el complejo mundo de la compilación, el generador de código intermedio se erige como el alquimista que traduce las intenciones del programador en un lenguaje intermedio comprensible por la máquina. En esta exploración profunda, descubriremos la esencia del generador de código intermedio, su función esencial en el proceso de traducción y su relación con las fases previas y posteriores del compilador

Funcionamiento interno

El proceso de análisis léxico implica la creación de un autómata finito determinista (DFA) o un autómata finito no determinista (NFA) para reconocer patrones léxicos en el código fuente. Se definen reglas específicas en forma de expresiones regulares para cada tipo de token. Estas expresiones describen patrones como números enteros, identificadores válidos, operadores, etc. El analizador léxico aplica estas expresiones regulares para escanear el código fuente y detectar la presencia de tokens.

Funcionamiento Interno y Transformaciones

El optimizador de código aplica una serie de transformaciones al código intermedio o al código de máquina, manteniendo el comportamiento del programa pero mejorando su rendimiento. Estas transformaciones pueden incluir la eliminación de código muerto (instrucciones no alcanzables), la simplificación de expresiones, la reorganización de instrucciones para explotar las características de la caché y la reducción de operaciones redundantes.

Importancia en el Proceso de Compilación

El generador de código objeto actúa como el vínculo esencial entre el mundo de la programación y la ejecución física de las instrucciones por parte de la máquina. Transforma las abstracciones que comprende el programador en un lenguaje que la computadora puede interpretar y ejecutar, haciendo realidad las intenciones del programador.

Elementos revisados

Lexer

omponente del compilador o intérprete que identifica los símbolos escritos en un programa como válidos. Es el que reconoce las letras, palabras y símbolos.

Parser

Lee el flujo de tokens, comprueba que cumple con las reglas sintácticas y produce el Resumen Árbol de sintaxis: una estructura de datos que representa la estructura sintáctica subyacente del programa de entrada.

Expresiones regulares

Una expresión regular es una cadena de caracteres que es utilizada para describir o encontrar patrones dentro de otros strings, en base al uso de delimitadores y ciertas reglas de sintaxis.

Flexbison

Son herramientas que permiten crear analizadores léxicos para el desarrollo de compiladores y han demostrado ser útiles en muchas otras áreas.

Tokens

Es una agrupación de caracteres reconocidos por el analizador léxico que constituyen los símbolos con los que se forman las sentencias del lenguaje.
